What is Kubernetes?

Kubernetes (K8s) is an open-source container orchestration platform that automates the deployment, scaling, and management of containerized applications. Originally developed by Google and now maintained by the Cloud Native Computing Foundation (CNCF).

Key Benefits

  • Automatic Scaling: Scales applications up/down based on demand
  • Self-Healing: Automatically restarts failed containers
  • Load Balancing: Distributes traffic across healthy containers
  • Declarative Configuration: Define desired state, K8s maintains it
  • Rolling Updates: Zero-downtime deployments
  • Service Discovery: Automatic networking between services
  • Storage Orchestration: Automatic mounting of storage systems
  • Configuration Management: Manage secrets and configurations
Why Kubernetes?
As applications grow from single containers to hundreds of microservices, manual management becomes impossible. Kubernetes provides the automation and abstractions needed to manage complex containerized applications at scale.

Kubernetes Architecture

Kubernetes follows a master-worker architecture with a control plane managing multiple worker nodes.

🎛️

Control Plane

The brain of the cluster that makes global decisions and responds to cluster events

🏭

Worker Nodes

Machines that run your application containers and provide the runtime environment

📦

Pods

Smallest deployable units containing one or more containers

🔄

Services

Stable network endpoints that expose applications to internal/external traffic

Core Components

Control Plane Components

API Server (kube-apiserver)

  • Frontend for the Kubernetes control plane
  • Exposes the Kubernetes REST API
  • All communication goes through the API server
  • Validates and processes API requests

Scheduler (kube-scheduler)

  • Assigns pods to worker nodes
  • Considers resource requirements and constraints
  • Factors in policies, affinity rules, and data locality
  • Makes optimal placement decisions

Controller Manager (kube-controller-manager)

  • Runs controller processes that regulate cluster state
  • Node Controller: monitors node health
  • Job Controller: manages one-off tasks
  • Endpoints Controller: populates service endpoints
  • Service Account & Token Controllers: manage accounts and API access tokens

etcd

  • Consistent and highly-available key-value store
  • Stores all cluster configuration data
  • Single source of truth for cluster state
  • Supports clustering for high availability

Worker Node Components

kubelet

  • Primary node agent running on each worker node
  • Communicates with the API server
  • Ensures containers are running in pods
  • Reports node and pod status
  • Manages container lifecycle

Container Runtime

  • Software responsible for running containers
  • Docker, containerd, CRI-O, or other CRI-compatible runtime
  • Pulls container images and runs containers
  • Manages container lifecycle and resources

kube-proxy

  • Network proxy running on each node
  • Maintains network rules for services
  • Enables communication between pods and services
  • Implements service load balancing
Component Interaction: All components work together to maintain desired cluster state. The API server acts as the central hub, while controllers continuously monitor and adjust the actual state to match the desired state.

Core Concepts

Pods

The smallest deployable unit in Kubernetes. A pod represents a single instance of a running process.

  • Contains one or more containers (usually one)
  • Containers share storage and network
  • Pods are ephemeral and replaceable
  • Each pod gets a unique IP address
# Simple pod example
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
spec:
  containers:
  - name: nginx
    image: nginx:1.21
    ports:
    - containerPort: 80

Deployments

Higher-level abstraction that manages a set of replica pods.

  • Ensures desired number of pod replicas
  • Provides rolling updates and rollbacks
  • Self-healing: replaces failed pods
  • Declarative updates to pods and ReplicaSets
# Deployment example
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.21
        ports:
        - containerPort: 80

Services

Stable network endpoint for accessing pods.

  • Provides load balancing across pods
  • Stable IP and DNS name
  • Service discovery mechanism
  • Types: ClusterIP, NodePort, LoadBalancer, ExternalName
# Service example
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
  type: ClusterIP

Namespaces

Virtual clusters within a physical cluster for resource isolation.

  • Isolate resources and applications
  • Organize cluster resources
  • Apply policies and resource quotas
  • Multi-tenancy support

ConfigMaps and Secrets

ConfigMaps: Store non-confidential configuration data

Secrets: Store sensitive information like passwords, tokens, and keys

Important: Secrets are base64 encoded, not encrypted. Use additional security measures for production environments.

Getting Started

Learning Path

  1. Set up a local cluster using minikube or kind
  2. Learn kubectl basic commands
  3. Deploy your first pod and service
  4. Create deployments with multiple replicas
  5. Explore networking and service types
  6. Work with volumes and persistent storage
  7. Manage configurations with ConfigMaps and Secrets
  8. Implement monitoring and logging

Essential kubectl Commands

# Cluster info
kubectl cluster-info
kubectl get nodes

# Working with pods
kubectl get pods
kubectl describe pod POD_NAME
kubectl logs POD_NAME
kubectl exec -it POD_NAME -- /bin/bash

# Working with deployments
kubectl create deployment nginx --image=nginx
kubectl get deployments
kubectl scale deployment nginx --replicas=3
kubectl rollout status deployment nginx

# Working with services
kubectl expose deployment nginx --port=80 --type=ClusterIP
kubectl get services
kubectl port-forward service/nginx 8080:80

# Working with namespaces
kubectl get namespaces
kubectl create namespace my-namespace
kubectl config set-context --current --namespace=my-namespace

# Applying configurations
kubectl apply -f deployment.yaml
kubectl delete -f deployment.yaml
