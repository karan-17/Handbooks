Launch EC2 Instance

Security Groups Configuration

# Create security group for web applications aws ec2 create-security-group \ --group-name docker-web-sg \ --description "Security group for Docker web applications" # Allow SSH access (replace YOUR_IP with your actual IP) aws ec2 authorize-security-group-ingress \ --group-name docker-web-sg \ --protocol tcp \ --port 22 \ --cidr YOUR_IP/32 # Allow HTTP traffic aws ec2 authorize-security-group-ingress \ --group-name docker-web-sg \ --protocol tcp \ --port 80 \ --cidr 0.0.0.0/0 # Allow HTTPS traffic aws ec2 authorize-security-group-ingress \ --group-name docker-web-sg \ --protocol tcp \ --port 443 \ --cidr 0.0.0.0/0
Security Best Practice

Never use 0.0.0.0/0 for SSH access. Always restrict to specific IP addresses or IP ranges. For production, consider using AWS Systems Manager Session Manager instead of direct SSH.

2
Connect to Instance
# Connect to EC2 instance ssh -i "your-key.pem" ubuntu@your-ec2-public-ip # Update system packages sudo apt update && sudo apt upgrade -y # Install essential packages sudo apt install -y curl wget git htop nano unzip
Connection Troubleshooting

If you can't connect, check: 1) Security group allows SSH from your IP, 2) Key file permissions are 400, 3) Using correct username (ubuntu for Ubuntu AMI), 4) Instance is in running state.

3. Docker Installation & Configuration

1
Install Docker Engine
# Remove any old Docker versions sudo apt remove -y docker docker-engine docker.io containerd runc # Install prerequisites sudo apt install -y \ ca-certificates \ curl \ gnupg \ lsb-release # Add Docker's official GPG key sudo mkdir -p /etc/apt/keyrings curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \ sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg # Add Docker repository echo \ "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \ https://download.docker.com/linux/ubuntu \ $(lsb_release -cs) stable" | \ sudo tee /etc/apt/sources.list.d/docker.list > /dev/null # Install Docker Engine sudo apt update sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
2
Configure Docker
# Start and enable Docker service sudo systemctl start docker sudo systemctl enable docker # Add ubuntu user to docker group (avoid using sudo) sudo usermod -aG docker ubuntu # Apply group changes (logout/login or use newgrp) newgrp docker # Verify installation docker --version docker compose version # Test Docker installation docker run hello-world
Installation Verification

If you see "Hello from Docker!" message, Docker is installed correctly and ready for deployment.

3
Production Docker Configuration
# Create Docker daemon configuration sudo mkdir -p /etc/docker # Configure Docker daemon for production sudo tee /etc/docker/daemon.json > /dev/null <
Configuration Explained
  • log-driver: JSON file logging with rotation
  • storage-driver: overlay2 for better performance
  • live-restore: Keep containers running during daemon restart
  • userland-proxy: Disable for better performance
  • icc: Disable inter-container communication by default

4. Security Configuration

1
Configure UFW Firewall
# Enable UFW firewall sudo ufw --force enable # Allow SSH (adjust port if changed) sudo ufw allow 22 # Allow HTTP and HTTPS sudo ufw allow 80 sudo ufw allow 443 # Deny all other incoming traffic by default sudo ufw default deny incoming sudo ufw default allow outgoing # Check firewall status sudo ufw status verbose
2
Configure Fail2Ban
# Install Fail2Ban for SSH protection sudo apt install -y fail2ban # Create local configuration sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local # Configure SSH protection sudo tee /etc/fail2ban/jail.local > /dev/null <
3
Docker Security Best Practices

Security Checklist

  • Run containers as non-root user
  • Use specific image tags, not 'latest'
  • Scan images for vulnerabilities
  • Limit container resources (CPU, memory)
  • Use secrets management for sensitive data
  • Enable Docker Content Trust
  • Regular security updates
  • Monitor container activities
