Shell Scripting Basics

Basic Shell Script Example
#!/bin/bash

# Variables
NAME="DevOps"
DATE=$(date +%Y-%m-%d)
LOG_FILE="/var/log/script.log"

# Functions
log_message() {
    echo "$(date): $1" >> $LOG_FILE
}

# Conditional statements
if [ -f "$LOG_FILE" ]; then
    log_message "Log file exists"
else
    touch "$LOG_FILE"
    log_message "Created new log file"
fi

# Loops
for i in {1..5}; do
    echo "Processing item $i"
    log_message "Processed item $i"
done

# Arrays
SERVICES=("nginx" "docker" "postgresql")
for service in "${SERVICES[@]}"; do
    if systemctl is-active "$service" >/dev/null 2>&1; then
        echo "$service is running"
    else
        echo "$service is not running"
    fi
done

# Error handling
set -e  # Exit on error
trap 'echo "Script failed at line $LINENO"' ERR

# Command line arguments
if [ $# -eq 0 ]; then
    echo "Usage: $0 "
    exit 1
fi

ENVIRONMENT=$1
echo "Deploying to $ENVIRONMENT environment"
⚠️ Important: Always test scripts in a safe environment before running in production. Use bash -n script.sh to check syntax without execution.

Cloud Service Models

🏗️ Infrastructure as a Service (IaaS)

What you manage: OS, runtime, middleware, applications, data

Provider manages: Virtualization, servers, storage, networking

Examples: AWS EC2, Google Compute Engine, Azure Virtual Machines

🚀 Platform as a Service (PaaS)

What you manage: Applications and data

Provider manages: Runtime, middleware, OS, virtualization, servers, storage, networking

Examples: AWS Elastic Beanstalk, Google App Engine, Azure App Service

📱 Software as a Service (SaaS)

What you manage: User access and data input

Provider manages: Everything else

Examples: Gmail, Salesforce, Microsoft 365, Slack

Cloud Adoption Strategy

The 6 R's of Migration

Migration Strategies
1. Rehost (Lift and Shift)
   - Move applications as-is to cloud
   - Fastest migration, minimal optimization
   - Example: VM-to-VM migration

2. Replatform (Lift, Tinker, and Shift)  
   - Minor optimizations during migration
   - Replace databases with managed services
   - Example: Self-managed MySQL → Amazon RDS

3. Refactor/Re-architect
   - Redesign for cloud-native architecture
   - Significant development effort
   - Example: Monolith → Microservices

4. Repurchase
   - Move to different product (usually SaaS)
   - Example: On-premises email → Office 365

5. Retire
   - Shut down unnecessary applications
   - Reduce complexity and costs

6. Retain
   - Keep on-premises for now
   - Revisit in future iterations

Multi-Cloud vs Hybrid Cloud

Multi-Cloud: Using multiple cloud providers simultaneously to avoid vendor lock-in and leverage best-of-breed services.

Hybrid Cloud: Combination of on-premises infrastructure with public cloud services, connected through dedicated networks.

Docker Fundamentals

What is Docker?

Docker is a platform that uses containerization to package applications and their dependencies into lightweight, portable containers that can run consistently across different environments.

🏗️ Containers vs VMs

  • Containers: Share OS kernel, lightweight, fast startup
  • VMs: Full OS per instance, heavier, slower startup
  • Containers: Process-level isolation
  • VMs: Hardware-level isolation

🎯 Docker Benefits

  • Consistent environments (dev/staging/prod)
  • Faster deployment and scaling
  • Resource efficiency
  • Microservices architecture enablement
  • Version control for infrastructure

Docker Security Best Practices

🔒 Image Security

  • Use official base images from trusted registries
  • Scan images for vulnerabilities
  • Use specific image tags, not 'latest'
  • Keep base images updated
  • Use minimal base images (alpine, distroless)

🛡️ Container Security

  • Run containers as non-root user
  • Use read-only filesystems when possible
  • Limit container capabilities
  • Use security profiles (AppArmor, SELinux)
  • Set resource limits (CPU, memory)

🌐 Network Security

  • Use custom networks for isolation
  • Don't expose unnecessary ports
  • Use TLS for inter-service communication
  • Implement network policies
  • Monitor network traffic

🔐 Secret Management

  • Never embed secrets in images
  • Use Docker secrets or external secret managers
  • Mount secrets as files, not environment variables
  • Rotate secrets regularly
  • Use least-privilege access to secrets

pipeline {
    agent any
    
    environment {
        DOCKER_REGISTRY = 'your-registry.com'
        IMAGE_NAME = 'webapp'
        KUBECONFIG = credentials('kubernetes-config')
    }
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
                script {
                    env.GIT_COMMIT_SHORT = sh(
                        script: 'git rev-parse --short HEAD',
                        returnStdout: true
                    ).trim()
                }
            }
        }
        
        stage('Build') {
            parallel {
                stage('Install Dependencies') {
                    steps {
                        sh 'npm ci'
                    }
                }
                stage('Security Scan') {
                    steps {
                        sh 'npm audit --audit-level high'
                        sh 'docker run --rm -v $(pwd):/app securecodewarrior/docker-security-scanner'
                    }
                }
            }
        }
        
        stage('Test') {
            parallel {
                stage('Unit Tests') {
                    steps {
                        sh 'npm run test:unit'
                    }
                    post {
                        always {
                            publishTestResults testResultsPattern: 'test-results/unit/*.xml'
                            publishCoverage adapters: [
                                istanbulCoberturaAdapter('coverage/cobertura-coverage.xml')
                            ], sourceFileResolver: sourceFiles('STORE_LAST_BUILD')
                        }
                    }
                }
                stage('Integration Tests') {
                    steps {
                        sh 'docker-compose -f docker-compose.test.yml up --build --abort-on-container-exit'
                    }
                    post {
                        always {
                            sh 'docker-compose -f docker-compose.test.yml down -v'
                        }
                    }
                }
                stage('Code Quality') {
                    steps {
                        withSonarQubeEnv('SonarQube') {
                            sh 'sonar-scanner'
                        }
                        timeout(time: 10, unit: 'MINUTES') {
                            waitForQualityGate abortPipeline: true
                        }
                    }
                }
            }
        }
        
        stage('Build Docker Image') {
            steps {
                script {
                    def image = docker.build("${env.DOCKER_REGISTRY}/${env.IMAGE_NAME}:${env.GIT_COMMIT_SHORT}")
                    docker.withRegistry("https://${env.DOCKER_REGISTRY}", 'docker-registry-creds') {
                        image.push()
                        image.push('latest')
                    }
                }
            }
        }
        
        stage('Security Scan Image') {
            steps {
                sh """
                    docker run --rm -v /var/run/docker.sock:/var/run/docker.sock \
                        aquasec/trivy:latest image \
                        --exit-code 1 --severity HIGH,CRITICAL \
                        ${env.DOCKER_REGISTRY}/${env.IMAGE_NAME}:${env.GIT_COMMIT_SHORT}
                """
            }
        }
        
        stage('Deploy') {
            when {
                anyOf {
                    branch 'main'
                    branch 'develop'
                }
            }
            steps {
                script {
                    def environment = env.BRANCH_NAME == 'main' ? 'production' : 'staging'
                    
                    sh """
                        helm upgrade --install ${env.IMAGE_NAME} ./helm-chart \
                            --namespace ${environment} \
                            --set image.repository=${env.DOCKER_REGISTRY}/${env.IMAGE_NAME} \
                            --set image.tag=${env.GIT_COMMIT_SHORT} \
                            --set environment=${environment} \
                            --wait --timeout=600s
                    """
                }
            }
        }
        
        stage('Post-Deployment Tests') {
            when {
                anyOf {
                    branch 'main'
                    branch 'develop'
                }
            }
            steps {
                script {
                    def environment = env.BRANCH_NAME == 'main' ? 'production' : 'staging'
                    def appUrl = environment == 'production' ? 
                        'https://myapp.com' : 'https://staging.myapp.com'
                    
                    sh """
                        # Smoke tests
                        curl -f ${appUrl}/health || exit 1
                        
                        # Run end-to-end tests
                        npm run test:e2e -- --base-url=${appUrl}
                    """
                }
            }
        }
    }
    
    post {
        always {
            cleanWs()
        }
        success {
            slackSend(

Blue-Green Pipeline Stage
stage('Blue-Green Deployment') {
    steps {
        script {
            // Get current active version
            def currentVersion = sh(
                script: "kubectl get service webapp -o jsonpath='{.spec.selector.version}'",
                returnStdout: true
            ).trim()
            
            def newVersion = currentVersion == 'blue' ? 'green' : 'blue'
            
            echo "Current version: ${currentVersion}, deploying to: ${newVersion}"
            
            // Deploy to inactive environment
            sh """
                helm upgrade --install webapp-${newVersion} ./helm-chart \
                    --set image.tag=${env.GIT_COMMIT_SHORT} \
                    --set version=${newVersion} \
                    --set service.enabled=false
            """
            
            // Wait for deployment to be ready
            sh "kubectl wait --for=condition=available --timeout=300s deployment/webapp-${newVersion}"
            
            // Run smoke tests against new version
            sh """
                kubectl port-forward deployment/webapp-${newVersion} 8080:8080 &
                PF_PID=\$!
                sleep 5
                curl -f http://localhost:8080/health || (kill \$PF_PID && exit 1)
                kill \$PF_PID
            """
            
            // Switch traffic to new version
            sh """
                kubectl patch service webapp -p '{"spec":{"selector":{"version":"${newVersion}"}}}'
            """
            
            echo "Traffic switched to ${newVersion}"
            
            // Optional: Keep old version for quick rollback
            timeout(time: 10, unit: 'MINUTES') {
                input message: "Deployment successful. Delete old version?", ok: 'Yes'
            }
            
            sh "kubectl delete deployment webapp-${currentVersion}"
        }
    }

📖 Glossary

Essential terms and definitions for cloud computing and DevOps

API Gateway: A service that acts as an entry point for client requests to backend services, handling authentication, rate limiting, and request routing.
Auto Scaling: Automatically adjusting the number of compute resources based on demand to maintain performance and optimize costs.
Blue-Green Deployment: A deployment strategy using two identical production environments (blue and green) to minimize downtime and reduce risk.
Container Orchestration: Automated management of containerized applications, including deployment, scaling, networking, and availability.
DevOps: A set of practices that combines software development (Dev) and IT operations (Ops) to shorten the development lifecycle and provide continuous delivery.
Immutable Infrastructure: Infrastructure components that are never modified after deployment; instead, they are replaced with new versions.
Load Balancer: Distributes incoming network traffic across multiple servers to ensure no single server becomes overwhelmed.
Microservices: An architectural approach where applications are built as a collection of loosely coupled, independently deployable services.
Serverless: A cloud computing execution model where the cloud provider dynamically manages the allocation and provisioning of servers.
Infrastructure as Code (IaC): Managing and provisioning computing infrastructure through machine-readable definition files rather than manual processes.
