1. Multi-Agent Systems Overview

Multi-agent systems represent a paradigm shift in AI development, moving from single, monolithic AI models to collaborative networks of specialized agents. This handbook covers three leading frameworks that enable the creation, orchestration, and deployment of multi-agent systems for various use cases.

Key Benefits of Multi-Agent Systems: Specialized expertise, scalable collaboration, fault tolerance, human-like problem-solving approaches, and the ability to handle complex, multi-step workflows that exceed the capabilities of single agents.

Framework Overview

2. CrewAI: Role-Playing Autonomous AI Agents

CrewAI is an open-source framework designed for orchestrating role-playing, autonomous AI agents. Built from scratch with no dependencies on LangChain, CrewAI offers faster execution and lighter resource demands while enabling collaborative intelligence through structured, role-based workflows.

Core Concepts

Agents

Specialized team members with specific skills, expertise, and responsibilities. Each agent has:

  • Role: What the agent does (e.g., "Senior Research Analyst")
  • Goal: What the agent optimizes for (e.g., "Uncover cutting-edge developments")
  • Backstory: Context that shapes behavior and expertise
  • Tools: Skills and functions the agent can utilize

Tasks

Individual assignments with clear descriptions, expected outputs, and assigned agents. Tasks support structured outputs using Pydantic and can reference outputs from previous tasks for complex workflows.

Crews

Collaborative groups of agents working together to achieve tasks. Crews define execution strategy, collaboration patterns, and overall workflow orchestration.

Python Implementation

from crewai import Agent, Task, Crew, Process
from crewai_tools import SerperDevTool
import os

# Set up environment
os.environ['OPENAI_API_KEY'] = 'your-api-key'

# Initialize tools
search_tool = SerperDevTool()

# Create agents
researcher = Agent(
    role='Senior Research Analyst',
    goal='Research and analyze information about given topics',
    backstory='Expert researcher with years of experience in data analysis',
    verbose=True,
    allow_delegation=False,
    tools=[search_tool]
)

writer = Agent(
    role='Content Writer',
    goal='Create engaging and informative content',
    backstory='Skilled writer who can transform research into compelling narratives',
    verbose=True
)

# Define tasks
research_task = Task(
    description='Research the latest trends in {topic}',
    expected_output='A detailed research report with key findings',
    agent=researcher
)

writing_task = Task(
    description='Write a blog post based on the research about {topic}',
    expected_output='A well-structured blog post in markdown format',
    agent=writer,
    context=[research_task]
)

# Create and run crew
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, writing_task],
    process=Process.sequential,
    verbose=True
)

result = crew.kickoff(inputs={'topic': 'artificial intelligence'})
print(result)

Advanced Features

CrewAI 2025: Flows Architecture

CrewAI 2025 introduced Flows for enterprise and production architecture:

from crewai import Flow, Agent, Task, Crew
from crewai.flow.flow import listen, router, start
from pydantic import BaseModel

class ProjectState(BaseModel):
    requirements: str = ""
    analysis: dict = {}
    implementation: str = ""
    status: str = "pending"

class SoftwareDevelopmentFlow(Flow[ProjectState]):
    
    @start()
    def gather_requirements(self) -> str:
        return "analyze_requirements"
    
    @listen(gather_requirements)
    def analyze_requirements(self, state: ProjectState) -> str:
        # Analysis crew
        analyst = Agent(
            role='Business Analyst',
            goal='Analyze and document requirements',
            backstory='Expert in requirement analysis'
        )
        
        analysis_task = Task(
            description='Analyze the requirements: {requirements}',
            expected_output='Detailed analysis with recommendations',
            agent=analyst
        )
        
        crew = Crew(agents=[analyst], tasks=[analysis_task])
        result = crew.kickoff(inputs={'requirements': state.requirements})
        state.analysis = result
        return "implementation_planning"
    
    @router(analyze_requirements)
    def decide_implementation(self, state: ProjectState) -> str:
        if state.analysis.get('complexity') == 'high':
            return "detailed_planning"
        return "direct_implementation"

# Usage
flow = SoftwareDevelopmentFlow()
result = flow.kickoff({"requirements": "Build a web application"})

Custom Tool Development

from crewai.tools import BaseTool
from typing import Type
from pydantic import BaseModel, Field
import requests

class WeatherInput(BaseModel):
    city: str = Field(..., description="City name to get weather for")

class WeatherTool(BaseTool):
    name: str = "Weather Tool"
    description: str = "Get current weather information for a city"
    args_schema: Type[BaseModel] = WeatherInput
    
    def _run(self, city: str) -> str:
        try:
            # Replace with actual weather API
            api_key = "your-weather-api-key"
            url = f"http://api.openweathermap.org/data/2.5/weather?q={city}&appid={api_key}"
            response = requests.get(url)
            data = response.json()
            
            if response.status_code == 200:
                temp = data['main']['temp']
                description = data['weather'][0]['description']
                return f"Weather in {city}: {description}, Temperature: {temp}K"
            else:
                return f"Could not get weather for {city}"
        except Exception as e:
            return f"Error getting weather: {str(e)}"

# Use custom tool with agent
weather_agent = Agent(
    role='Weather Analyst',
    goal='Provide accurate weather information',
    backstory='Expert meteorologist',
    tools=[WeatherTool()],
    verbose=True
)

3. Microsoft Agent Framework: Enterprise Multi-Agent Orchestration

Microsoft Agent Framework is an enterprise-grade, open-source SDK and runtime that simplifies the orchestration of multi-agent systems. It represents the convergence of AutoGen and Semantic Kernel into a unified, commercial-grade framework for building, orchestrating, and deploying AI agents.

Core Architecture

Multi-Agent Orchestration

  • Graph-Based Workflows: Streaming, checkpointing, human-in-the-loop capabilities
  • Enterprise Foundation: Session-based state management, type safety, middleware
  • Cross-Platform Support: Consistent APIs for Python and C#/.NET
  • Built-in Observability: OpenTelemetry integration for monitoring and debugging

Azure Integration

  • Azure AI Foundry: Seamless cloud deployment with compliance
  • Microsoft 365: Integration with Copilot and Office platforms
  • Teams Integration: Build agents for Microsoft Teams collaboration
  • Azure Services: Native integration with AI services and Graph API

4. AutoGen: Multi-Agent Conversation Systems

⚠️ Important: AutoGen Migration Notice

AutoGen is entering maintenance mode and will only receive bug fixes and security patches. Microsoft recommends migrating to Microsoft Agent Framework for new projects. Existing AutoGen projects should plan migration within 6-12 months.

Microsoft AutoGen is a programming framework for agentic AI that enables multi-agent conversation systems. AutoGen v0.4 features an asynchronous, event-driven architecture where agents communicate through asynchronous messages.

Core Concepts

Multi-Agent Conversation Frameworks

  • Dynamic Conversations: Agent topology adapts based on conversation flow
  • Static Conversations: Adhere to predefined topology for structured workflows
  • Group Chat: Multiple agents in single conversation thread with shared context
  • Hierarchical Patterns: Support for nested chats and complex orchestration

Agent Types

AssistantAgent

General-purpose agent for prototyping and education

  • Language model integration
  • Tool capabilities
  • Multi-modal support

UserProxyAgent

Simulates human behavior with configurable intervention

  • User input handling
  • Autonomous or interactive modes
  • Code execution capabilities

Specialized Agents

Domain-specific agents for particular tasks

  • CodeExecutorAgent
  • MultimodalWebSurfer
  • FileSurfer, VideoSurfer

    "coder",

    "user_proxy", 
    code_execution_config={"executor": "local"}
)

    messages=[],
    max_round=10

    manager, 
    message="Research and create a data visualization for climate change trends."
)

Advanced Group Chat with State Transitions

# Custom speaker selection logic
def state_transition(last_speaker, groupchat):
    messages = groupchat.messages
    
    if last_speaker is researcher:
        return coder  # Researcher passes to coder
    elif last_speaker is coder:
        return user_proxy  # Coder passes to user for execution
    else:
        return researcher  # User passes back to researcher

group_chat = GroupChat(
    agents=[user_proxy, researcher, coder],
    messages=[],
    max_round=15,
    speaker_selection_method=state_transition
)

Use Case Recommendations

Choose CrewAI When:

  • Rapid Prototyping: Need to quickly test multi-agent concepts
  • Role-Based Workflows: Clear agent roles and responsibilities
  • Structured Processes: Sequential or hierarchical task execution
  • Educational Projects: Learning multi-agent development
  • Startup/SMB: Cost-effective solution with good capabilities

Choose Microsoft Agent Framework When:

  • Enterprise Deployment: Production-grade requirements
  • Azure Ecosystem: Heavy Microsoft technology usage
  • Compliance Needs: Regulatory and security requirements
  • Scale Requirements: Large-scale, distributed deployments
  • Long-term Investment: Strategic Microsoft partnership

Choose AutoGen When:

  • Research Projects: Academic or experimental work
  • Existing Investment: Already using AutoGen extensively
  • Short-term Projects: 6-month or shorter timelines
  • Conversation Focus: Heavy emphasis on chat patterns
  • ⚠️ With Migration Plan: Plan transition to Agent Framework

)

research_task = Task(
    description='Research latest trends and insights about {topic}',

)

)

# Execute the workflow
result = content_crew.kickoff(inputs={'topic': 'Sustainable Technology Trends 2025'})
print(result)

    role='Technical Support Specialist',
    goal='Resolve technical issues and provide solutions',
    backstory='Senior technical support engineer with troubleshooting expertise',
    tools=[SerperDevTool(), FileReadTool()],
    verbose=True

    role='Escalation Manager',
    goal='Handle complex issues requiring human intervention',
    backstory='Experienced manager who determines escalation needs',
    verbose=True
)

# Customer Support Flow
class CustomerSupportFlow(Flow):
    
    @start()
    def classify_inquiry(self, customer_message: str) -> str:
        # Classify the customer inquiry
        task = Task(
            description=f'Classify this customer inquiry: {customer_message}',
            expected_output='Classification: TECHNICAL, BILLING, GENERAL, ESCALATION',
            agent=classifier
        )
        
        result = task.execute()
        self.classification = result
        return "route_to_specialist"
    
    @router("route_to_specialist")
    def route_inquiry(self) -> str:
        if "TECHNICAL" in self.classification:
            return "handle_technical"
        elif "ESCALATION" in self.classification:
            return "escalate_issue"
        else:
            return "handle_general"
    
    @listen("handle_technical")
    def handle_technical_issue(self) -> str:
        task = Task(
            description=f'Resolve this technical issue: {self.customer_message}',
            expected_output='Technical solution with step-by-step instructions',
            agent=technical_support
        )
        
        self.resolution = task.execute()
        return "complete"

# Usage

    "customer_message": "My application keeps crashing when I try to export data"

7. Migration Paths and Best Practices

As the multi-agent landscape evolves, understanding migration paths between frameworks becomes crucial for long-term project success.

AutoGen to Microsoft Agent Framework Migration

⚠️ Critical Migration Timeline

With AutoGen entering maintenance mode, existing AutoGen projects should plan migration to Microsoft Agent Framework within 6-12 months to avoid technical debt accumulation and access new enterprise capabilities.

Migration Strategy

Phase 1: Assessment (Month 1-2)
  1. Inventory Analysis: Document existing AutoGen agents and workflows
  2. Feature Mapping: Identify Agent Framework equivalents
  3. Infrastructure Review: Assess Azure integration requirements
  4. Training Planning: Identify team training needs
Phase 2: Proof of Concept (Month 2-3)
  1. Parallel Development: Recreate key workflows in Agent Framework
  2. Performance Testing: Compare execution and resource usage
  3. Integration Testing: Validate Azure service integrations
  4. Team Training: Begin Agent Framework education
Phase 3: Gradual Migration (Month 4-8)
  1. Component-by-Component: Migrate non-critical workflows first
  2. Parallel Running: Run both systems during transition
  3. Monitoring Setup: Implement comprehensive observability
  4. User Acceptance: Validate functionality with stakeholders
Phase 4: Complete Transition (Month 9-12)
  1. Critical Workloads: Migrate remaining production systems
  2. AutoGen Decommission: Safely retire AutoGen infrastructure
  3. Documentation: Update all technical documentation
  4. Knowledge Transfer: Complete team training and certification
