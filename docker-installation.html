Installation Overview

🪟 Windows Installation

Docker Desktop with WSL2 backend for optimal performance. Includes Hyper-V setup, system requirements, and complete configuration guide.

System Requirements

  • Windows 10/11 64-bit (Build 19041 or later)
  • WSL 2 enabled with virtualization support
  • 4 GB RAM minimum (8 GB recommended)
  • BIOS virtualization enabled (VT-x/AMD-V)

🍎 macOS Installation

Native Docker Desktop for Intel and Apple Silicon Macs. Optimized for M1/M2/M3 chips with Rosetta 2 considerations.

System Requirements

  • macOS Monterey (12) or later
  • 4 GB RAM minimum (8 GB recommended)
  • Intel or Apple Silicon processor
  • Apple Virtualization Framework support

🐧 Linux Installation

Docker Engine and Docker Desktop options. Support for Ubuntu, Debian, CentOS, Fedora, and other major distributions.

System Requirements

  • 64-bit Linux distribution
  • Kernel version 3.10 or higher
  • 2 GB RAM minimum
  • SystemD or compatible init system

🪟 Windows Installation with WSL2

💡
2026 Update: Docker Desktop now requires WSL2 for optimal performance on Windows. This guide covers the complete setup process including WSL2 configuration.

Pre-Installation: Enable WSL2

  1. Check System Compatibility
    PowerShell (Run as Administrator)
    # Check Windows version
winver

# Check virtualization support
systeminfo | findstr /C:"Hyper-V"

# Verify system information
Get-ComputerInfo | Select-Object WindowsProductName, WindowsVersion
  2. Enable Windows Features
    PowerShell (Run as Administrator)
    # Enable WSL
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

# Enable Virtual Machine Platform
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

# Restart Windows (Required)
Restart-Computer
  3. Configure WSL2 After Restart
    PowerShell
    # Set WSL2 as default version
wsl --set-default-version 2

# Update WSL to latest version
wsl --update

# Install Ubuntu (or preferred distribution)
wsl --install -d Ubuntu

# Check WSL status
wsl --status

Install Docker Desktop

  1. Download Docker Desktop

    Download the latest Docker Desktop for Windows from the official Docker website or use the link above.

  2. Run Installation
    Command Options
    # Silent installation with WSL2 backend
Start-Process -FilePath "Docker Desktop Installer.exe" -ArgumentList "install --quiet --accept-license --backend=wsl-2" -Wait

# Standard installation (GUI)
# Double-click Docker Desktop Installer.exe
# Keep "Use WSL 2 instead of Hyper-V" checked
  3. Configure WSL Integration

    After installation, open Docker Desktop Settings → Resources → WSL Integration and enable your WSL distributions.

Common Windows Installation Issues

WSL Context Errors

If you see deprecated "wsl" context errors:

Fix Commands
# Check contexts
docker context ls

# Remove deprecated context
docker context rm wsl

# Use default context
docker context use default
Virtualization Not Enabled

Error: "Hardware acceleration is not available"

  1. Restart computer and enter BIOS/UEFI settings
  2. Enable Intel VT-x or AMD-V virtualization
  3. Enable Hyper-V if using Hyper-V backend
  4. Save and restart
WSL Access Rights Issues

Docker shows "access rights not set correctly" warning:

# Restart Docker Desktop

Performance Optimization

🚀 macOS Performance Tips

🔧 Virtualization Framework

Use Apple's Virtualization Framework instead of HyperKit for better performance on Apple Silicon.

📁 File Sharing

Use VirtioFS for faster file operations between host and containers, especially for large codebases.

💾 Resource Limits

Allocate appropriate CPU and memory limits. Apple Silicon can handle higher limits efficiently.

🏗️ Multi-Platform Builds

Leverage buildx for building images that work on both ARM64 and x86_64 architectures.

🐧 Linux Installation

🐧
Linux Options: Choose between Docker Desktop (GUI experience) or Docker Engine (lightweight, server-ideal). Both support the same container workflows.

Ubuntu Installation

  1. Remove Old Docker Packages
    Terminal
    # Remove conflicting packages
sudo apt-get remove docker docker-engine docker.io containerd runc

# Clean up old configurations
sudo apt-get purge docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
  2. Set Up Docker Repository
    Terminal
    # Update package index
sudo apt-get update

# Install prerequisites
sudo apt-get install ca-certificates curl gnupg lsb-release

# Add Docker's official GPG key
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Set up repository
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
  3. Install Docker Engine
    Terminal
    # Update package index
sudo apt-get update

# Install Docker Engine, CLI, and plugins
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Verify installation
sudo docker run hello-world
  4. Post-Installation Steps
    Terminal
    # Add user to docker group (avoid sudo)
sudo usermod -aG docker $USER

# Enable Docker service
sudo systemctl enable docker.service
sudo systemctl enable containerd.service

# Apply group changes (logout/login or use newgrp)
newgrp docker

# Test without sudo
docker run hello-world

CentOS/RHEL Installation

Terminal
# Remove old Docker packages
sudo yum remove docker docker-client docker-client-latest docker-common docker-latest docker-latest-logrotate docker-logrotate docker-engine

# Install prerequisites
sudo yum install -y yum-utils

# Set up repository
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# Install Docker Engine
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Start and enable Docker
sudo systemctl start docker
sudo systemctl enable docker

# Add user to docker group
sudo usermod -aG docker $USER

Debian Installation

Terminal
# Remove old packages
sudo apt-get remove docker docker-engine docker.io containerd runc

# Update and install prerequisites
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg lsb-release

# Add Docker's GPG key and repository
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker Engine
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

Linux Installation Troubleshooting

Permission Denied Socket Errors

Cannot connect to Docker daemon socket:

Fix Commands
# Check Docker service status
sudo systemctl status docker

# Start Docker service
sudo systemctl start docker

# Add user to docker group
sudo usermod -aG docker $USER

# Apply group changes
newgrp docker

# Fix socket permissions
sudo chown root:docker /var/run/docker.sock
sudo chmod 660 /var/run/docker.sock
Systemd Service Issues

Docker service fails to start:

Diagnostic Commands

sudo journalctl -u docker.service

# Reset service
sudo systemctl daemon-reload
sudo systemctl restart docker

# Check for conflicts
sudo systemctl list-units --failed

Compose Version Migration

🔄
V1 to V2 Migration: Docker Compose v2 uses docker compose (space) instead of docker-compose (hyphen). The functionality is identical.
Version Comparison
# V1 Syntax (deprecated)
docker-compose up -d
docker-compose ps
docker-compose down

# V2 Syntax (current)
docker compose up -d
docker compose ps
docker compose down

# Both work the same way!

✅ Installation Verification & Testing

Basic Verification Steps

  1. Check Docker Version
    Terminal
    # Check Docker version
docker --version

# Get detailed version information
docker version

# Check system information
docker system info
  2. Run Hello World Container
    Terminal
    # Test basic Docker functionality
docker run hello-world

# Expected output includes:
# "Hello from Docker! This message shows that your installation appears to be working correctly."
  3. Test Container Operations
    Terminal
    # Run an interactive container
docker run -it ubuntu:latest /bin/bash

# In the container, run some commands:
# cat /etc/os-release
# exit

# Check running containers
docker ps

# Check all containers
docker ps -a
  4. Test Docker Compose
    Terminal
    # Check Compose version
docker compose version

# Create a simple test file
echo 'version: "3.8"
services:
  test:
    image: nginx:alpine
    ports:
      - "8080:80"' > docker-compose.test.yml

# Test Compose functionality
docker compose -f docker-compose.test.yml up -d
docker compose -f docker-compose.test.yml ps
docker compose -f docker-compose.test.yml down

# Clean up
rm docker-compose.test.yml

🔍 Troubleshooting Common Issues

Universal Issues (All Platforms)

Docker Daemon Not Running

Symptoms: "Cannot connect to the Docker daemon" error

Solutions
# Check if Docker is running
docker version

# Start Docker service (Linux)
sudo systemctl start docker

# Start Docker Desktop (Windows/macOS)
# Launch from Applications or Start Menu

# Check Docker status
docker system info
Permission Denied Errors

Symptoms: "permission denied while trying to connect to the Docker daemon socket"

Linux Fix
# Add user to docker group
sudo usermod -aG docker $USER

# Apply group changes
newgrp docker

# Or logout and login again
Container Won't Start

Symptoms: Container exits immediately or fails to start

Diagnostic Commands
# Check container logs
docker logs 

# Run container in interactive mode for debugging
docker run -it  /bin/sh

# Check container resource usage
docker stats

# Inspect container configuration
docker inspect
Port Already in Use

Symptoms: "bind: address already in use" error

Solutions
# Find process using port
sudo lsof -i :8080  # Replace 8080 with your port
netstat -tulpn | grep :8080

# Use different port
docker run -p 8081:80 nginx  # Use 8081 instead

# Stop conflicting service
sudo systemctl stop apache2  # Example

Windows-Specific Issues

WSL2 Backend Issues
WSL2 Fixes
# Update WSL2
wsl --update

# Reset WSL2 if corrupted
wsl --shutdown
wsl --unregister Ubuntu  # Or your distro name
wsl --install -d Ubuntu

# Check WSL integration in Docker Desktop settings
Hyper-V Conflicts

Issue: Virtualization conflicts with VirtualBox or VMware

PowerShell (Admin)
# Disable Hyper-V temporarily
bcdedit /set hypervisorlaunchtype off

# Re-enable Hyper-V
bcdedit /set hypervisorlaunchtype auto

# Restart required after changes

macOS-Specific Issues

File Sharing Performance

Issue: Slow file operations between host and containers

  1. Enable VirtioFS in Docker Desktop settings
  2. Use named volumes for better performance
  3. Avoid bind mounts for large directories
  4. Consider using Docker's file synchronization features
Apple Silicon Compatibility

Issue: x86 images won't run on Apple Silicon

Solutions
# Force x86 platform
docker run --platform linux/amd64 

# Build for specific platform
docker buildx build --platform linux/amd64 -t myapp:x86 .

# Install Rosetta 2 if needed
softwareupdate --install-rosetta

Linux-Specific Issues

Systemd Service Problems
Service Management
# Check service status
sudo systemctl status docker

# Restart Docker service
sudo systemctl restart docker

# Enable auto-start
sudo systemctl enable docker

# View service logs
sudo journalctl -u docker.service
Storage Driver Issues

Issue: Docker fails to start due to storage driver problems

Storage Fixes
# Check storage driver
docker info | grep "Storage Driver"

# Reset Docker data (WARNING: removes all containers/images)
sudo systemctl stop docker
sudo rm -rf /var/lib/docker
sudo systemctl start docker

🐳 Custom Development Dockerfile

.devcontainer/Dockerfile
FROM ubuntu:22.04

# Install development dependencies
RUN apt-get update && apt-get install -y \
    curl \
    git \
    vim \
    build-essential \
    python3 \
    python3-pip \
    nodejs \
    npm \
    && rm -rf /var/lib/apt/lists/*

# Install development tools
RUN npm install -g typescript @angular/cli

# Create development user
RUN useradd -m -s /bin/bash developer
USER developer
WORKDIR /home/developer

# Configure git (will be overridden by user)
RUN git config --global user.name "Developer" && \
    git config --global user.email "dev@example.com"
