Doc2Chart: Intent-Driven Zero-Shot Chart Generation

arXiv ID: 2507.14819
Authors: Akriti Jain, Pritika Ramu, Aparna Garimella, Apoorv Saxena
Publication: EMNLP 2025
Date: July 20, 2025 (v1), August 27, 2025 (v2)

Research Contribution

Doc2Chart introduces the novel task of intent-based chart generation from documents, addressing the challenge of applying Large Language Models (LLMs) to visualize data from long documents based on user-specified intents in a zero-shot setting.

Key Innovation

The first work to address document-to-chart generation based on intent in a zero-shot setting, proposing an unsupervised, two-staged framework that generates charts adhering to user intent and grounded in document content without requiring manual data pre-selection.

Technical Framework

Two-Stage Architecture:
  • Information Extraction Stage: Intent decomposition using LLMs, iterative data extraction and refinement process, validation mechanisms to ensure data fidelity
  • Chart Generation Stage: Heuristic-guided chart type selection, predefined visualization heuristics, final chart code generation
Novel Evaluation Methodology:
  • Attribution-based evaluation metric: Uses structured text representation of generated charts
  • Attention-based heat map analysis: Detects spans of data values not captured in reference charts
  • Reference-free evaluation: Assesses chart faithfulness against source documents directly

Relevance to Talking BI

  • Automated insight generation: Enables financial reports, business documents analysis
  • Natural language interface: Users can request specific charts using conversational intent
  • Document-grounded responses: Charts are factually grounded in source business documents
  • Zero-shot capability: No training required for new document types or domains

Performance Results

Tested on 4 LLMs (GPT-4o, Gemini-2.0, LLaMA-3.1-8B-Instruct, Claude-3.5-Sonnet) with a dataset of 1,242 validated tuples, showing significant improvements over baselines in appropriate chart type selection, chart data accuracy, and intent adherence.

Key Takeaways for Practitioners

  • Multi-stage processing is crucial for accurate document-to-chart generation
  • Intent decomposition is essential for handling complex user requests
  • Validation and refinement loops significantly improve output quality
  • Attribution mechanisms are necessary for ensuring data faithfulness

InsightEval: Expert-Curated Benchmark for Insight Discovery

arXiv ID: 2511.22884
Authors: Zhenghao Zhu, Yuanfeng Song, Xin Chen, Chengzhong Liu, Yakun Cui, Caleb Chen Cao, Sirui Han, Yike Guo
Publication Date: November 28, 2025

Research Problem

The paper addresses a critical gap in evaluating LLM-driven data analysis systems, noting that while "more and more researchers are making use of LLMs and multi-agent systems for insight discovery," there are insufficient benchmarks for evaluating these capabilities.

Main Contribution

Development of InsightEval, an expert-curated benchmark that provides a more robust evaluation framework than existing tools like InsightBench, which suffered from "format inconsistencies, poorly conceived objectives, and redundant insights."

Methodological Innovations

Data Curation Pipeline:
  • Comprehensive data-curation pipeline to construct the InsightEval dataset
  • Expert curation to ensure high-quality evaluation standards
  • Structured taxonomy development for table-driven insight generation
Novel Evaluation Metrics:
  • Novel metrics to measure exploratory performance of agents
  • Revised evaluation criteria addressing identified shortcomings
  • Integration of refined goals, tabular schema, questions, and answers

Performance Analysis

  • Insights F1 Scores: Consistently exceeded Insight Recall Scores across all tested agents
  • Human Alignment: F1 scores demonstrated closer alignment with Human Evaluation Scores
  • Agent Comparison: Agent Poirot (Claude 3.7 Sonnet) substantially outperformed other agents in G-Eval assessments

Business Intelligence Applications

  • Automated Insight Generation: Framework applies to BI systems that automatically discover insights from business data
  • Multi-Agent Analytics: Supports development of sophisticated BI systems using multiple specialized agents
  • Conversational Analytics: Enables evaluation of conversational AI systems providing business insights through natural language
  • Decision-making Support: Facilitates agentic workflows that are visible, editable, and explainable

Technical Innovations

  • Comprehensive Assessment Protocol: Standardized methodology for evaluating LLM-driven data agents
  • Multi-dimensional Evaluation: Assessment across accuracy, relevance, and novelty criteria
  • Human-AI Alignment: Metrics designed to align with human judgment in insight quality assessment
  • Cross-Agent Validation: Testing across multiple agent architectures and language models

Practitioner Recommendations

  • Use InsightEval as a benchmark for developing and evaluating conversational BI systems
  • Consider implementing specialized agents for different analytical tasks
  • Prioritize metrics that align with human judgment rather than purely algorithmic measures
  • Account for the subjective nature of insights when designing evaluation criteria

DeepPrep: LLM-Powered Autonomous Data Preparation

arXiv ID: 2602.07371
Authors: Meihao Fan, Ju Fan, Yuxin Zhang, Shaolei Zhang, Xiaoyong Du, Jie Song, Peng Li, Fuxin Jiang, Tieying Zhang, Jianjun Chen
Publication Date: February 7, 2026
Category: Computer Science - Artificial Intelligence

Problem Statement

Data preparation remains a major bottleneck in data science, requiring transformation of heterogeneous and noisy raw tables into analysis-ready data. Existing LLM-powered methods have two key limitations: they make decisions without grounding in intermediate execution results, and they rely on linear interaction processes that offer limited support for revising earlier decisions.

Core Innovation

DeepPrep introduces an LLM-powered agentic system that constructs data preparation pipelines through iterative, execution-grounded interaction with an environment that materializes intermediate table states and returns runtime feedback.

Technical Architecture

Tree-Based Agentic Reasoning:
  • Organizes pipeline construction using tree-based reasoning rather than linear approaches
  • Enables structured exploration and non-local revision based on execution feedback
  • Maintains explicit tree of materialized execution states during pipeline construction
Execution-Grounded Interaction:
  • Iterative interaction with environment that materializes intermediate table states
  • Returns runtime feedback for decision-making
  • Enables precise attribution of execution feedback to earlier decision points

Process Flow

  1. Planning: Observing current execution state and synthesizing high-level plans
  2. Execution: Applying operator pipelines and materializing intermediate results
  3. Feedback Integration: Using execution results to guide future decisions and enable backtracking

Performance Results

  • Accuracy: Comparable to strong closed-source models (e.g., GPT-5)
  • Cost Efficiency: 15x lower inference cost compared to closed-source alternatives
  • Open Source Performance: State-of-the-art among open-source baselines
  • Generalization: Effective across diverse datasets

Talking BI Integration

  • Conversational Data Preparation: Accepts natural language specifications for data transformation tasks
  • Interactive Refinement: Enables business users to specify requirements without technical expertise
  • Autonomous Quality Management: Automatically detects and corrects data quality issues
  • Transparent Processing: Provides explanations for transformation decisions and maintains audit trails

Technical Innovations

  • Tree-Based Pipeline Construction: First system to organize data preparation as tree search
  • Execution-Grounded Decision Making: Grounds decisions in actual execution results
  • Progressive Agentic Training: Novel training approach for autonomous data preparation
  • State Materialization: Explicit representation of intermediate execution states

Limitations and Future Work

  • Scalability: Tree-based approach may face challenges with very large datasets
  • Domain Specificity: Current focus on tabular data preparation
  • Future Directions: Extension to non-tabular data formats, enhanced reasoning capabilities

Business Value for Talking BI

  • Cost-Effective Solution: 15x cost reduction while maintaining accuracy
  • Reduced Technical Barriers: Enables natural language data preparation specifications
  • Quality Assurance: Systematic error detection and transparent audit trails
  • Reproducible Pipelines: Enables consistent, repeatable data preparation processes
